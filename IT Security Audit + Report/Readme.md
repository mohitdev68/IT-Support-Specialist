## Project Overview
Windows Security Audit · Built-in tools only 
A hands-on security audit of a Windows 11 system using only native tools. This project simulates the kind of systematic security review IT support specialists perform in real companies to verify systems meet baseline security standards.

What Is a Security Audit?
A security audit is a systematic check of a system's security posture — identifying what is configured correctly, what is weak, and what needs fixing. IT support specialists perform security audits regularly to ensure systems comply with organizational and industry security standards.

Audit Areas
Audit 1 — Windows Defender
Antivirus status, real-time protection, last scan
The most basic and fundamental check in any IT security audit. Windows Defender is the first layer of protection against malware and malicious software. This audit verifies that real-time protection is active, definitions are up to date, and that recent scans have been completed.

Audit 2 — Windows Firewall
Rules, active profiles, blocked connections
The firewall is the first line of defence against network-based threats. This audit examines which firewall profiles are active (Domain, Private, Public), reviews inbound and outbound rules, and checks for any anomalies or overly permissive rules that could expose the system to attack.

Audit 3 — User Account Permissions
Admin accounts, guest accounts, account status
Unauthorized administrator accounts are a critical security risk — they give attackers elevated access if compromised. This audit inventories all local user accounts, identifies which accounts hold admin privileges, checks whether the Guest account is disabled, and flags any dormant or suspicious accounts.

Audit 4 — Password Policies
Complexity, expiry, lockout settings
Weak password policies are the number one cause of security breaches. This audit reviews the local password policy including minimum length, complexity requirements, maximum password age, and account lockout thresholds to ensure they meet security best practices.

Audit 5 — Windows Updates
Missing updates, last update date, pending patches
Unpatched systems are the most common attack vector exploited in the wild. This audit checks the current patch level, identifies any missing or pending updates, and records when the last successful update occurred.

 Audit Report Summary
The final report documents findings across all five audit areas using a RAG (Red / Amber / Green) status system:
StatusMeaning🟢 PassConfigured correctly, meets baseline🟡 WarningNeeds attention or improvement🔴 FailCritical risk, immediate action required
